    App Features

    Authentication via social media
    Users can authenticate to the app with their google accounts
    Roles restrictions
    Different user roles restrictions, for example, possibility to perform actions like APPROVE OR REJECT for manager reviewer. That's why manager, which create order, don't see such buttons at all
    Notifications via email and FLP
    Users receive notification in FLP and via their email addresses
    PDF documents creation
    For WH Orders present two kinds of reports, which users can open as full WH Order summary report or for each order product as packaging label with QRCode
    Mobile QRCode scanner application
    Additional mobile QRCode scanner application allows WH packagers scan each QRCode on packaging label and trigger status changing
    Custom Elements and Visualizations
    Custom tiles for related WH Orders are present at Order Object page which show some delivery statistics and navigate to separate WH Order Object page. Also progress indicator with colored accents improve user experience

    Application Idea

    Org structure

    Lets realize some organization with several departments, in my case laboratories, warehouses and suppliers department. The main idea was to automate routine operations, which do suppliers managers, from placing new order to its delivery to department from warehouses.
    Main Flow
    After placing new order by manager its need to be approved by reviewer. If reviewer ask for order editing, manager need to do that and again receive approve. If reviewer reject the order, its automatically closes. After approve by reviewer order splits to several suborders - WH Orders.
    Main Flow
    Each WH Order has its own processor - WH packager, which collect products by scanning QRCodes on packaging labels and when all products are collected this order is send to department. Only when all of WH Orders are delivered, the main order is completed and successfully closed.
    Entity Relationships
    The main entity is Order, witch consists of WarehouseProducts from different Warehouses. Thats why this Order splits on suborders - Wh Orders, which by contract has products from only one WH. Therefore delivery takes place from WH address to Department address.
